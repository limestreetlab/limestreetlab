  • Java Programming
  • Joyce Farrell
  • A nice first beginner book on Java, but doesn't add value if already familiar with the basics.
    Easy to understand explanations of basic Java constructs and syntax as well as OO concepts, followed by short code examples.
    Compared to David Eck and Daniel Liang, this book is more introductory and so its coverage is solely focused on beginner topics (no lambdas, generic types, hash maps, binary trees, threads, etc) and writing style more verbose.
    Its discussion on files IO is sub-optimal as those byte stream-text stream bridge classes such as InputStreamReader and OutputStreamWriter are not properly introduced. For GUI, it would be better if he skips deprecated Swing altogether and just introduces JavaFX.

  • Introduction to Programming in Java
  • Robert Sedgewick
  • This book is really about the basics of programming, focusing on key basic concepts rather than actual codes. In fact, author imports his defined classes (I/O etc) in his examples to hide away unnecessary details in order to concentrate on concepts. The codes and topics of Java discussed are very basic but the examples are very good and illustrative in showing how such basic concepts and data types can be used as building blocks for something useful. Unlike Liang and Eck which cover a lot, this is appropriately focused yet most educational as long as the examples and exercises are thoroughly learnt. Best college textbook.

  • Objects First with Java
  • David Barnest
  • This isn't a college textbook and should be read after an introductory text like Eck and Sedgewick. It teaches fundamental OOP concepts, coding styles/practices, program designs, testing/debugging (test class/printing) using the BlueJ IDE (worth re-reading) and many sample projects (good and diverse). Chapter concepts are introduced through realworld cases (emails, facebook, predator-prey simulation etc) in an often progressive manner to highlight areas of improvement so the what and why are easier to understand. Author often emphasizes proper OOP styles (loose coupling, cohesion, encapsulation, documentation, reusability, maintainability, extendibility etc) to enforce good coding habits. Would be much better if GUI were taught in JavaFX.
